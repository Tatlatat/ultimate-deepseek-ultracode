**Goal:** Add CI: de-flake the one timing-raced test, add a `tests/run-all.sh` entry-point, and a GitHub Actions workflow that runs the offline suite on every PR/push.

- No behavior change to gateway/engine/levers. B1 only widens a test's timing margin (keeps the assertion); B2/B3 are new files (run-all.sh, ci.yml).
- Benches in `runtime/` are NOT in CI (need real DeepSeek). run-all.sh only touches `tests/`.
- The guard `tests/test-no-codex-leftovers.py` runs as part of run-all (gates every PR).

### Task 2: `tests/run-all.sh` entry-point (B2)

**Files:**
- Create: `tests/run-all.sh`

**Interfaces:**
- Produces: a script that runs every `tests/test-*.py` (python3), `tests/test-*.mjs` (node), `tests/test-*.sh` (bash, excluding run-all.sh), prints a per-file PASS/FAIL + summary, exits 0 if all pass / 1 if any fail.

- [ ] **Step 1: Write `tests/run-all.sh`**

```bash
#!/usr/bin/env bash
# Run the full CI-safe test suite (python + node + shell). Offline: no DeepSeek,
# no network — every test runs in mock mode. Exits non-zero if any test fails.
# Benches (runtime/*bench*.py) need real DeepSeek and are NOT run here.
set -u
cd "$(dirname "$0")/.." || exit 2
pass=0; fail=0; failed=()

run_one() {  # $1=label  $2..=command
  local label="$1"; shift
  if "$@" >/dev/null 2>&1; then
    echo "  ok   $label"; pass=$((pass+1))
  else
    echo "  FAIL $label"; fail=$((fail+1)); failed+=("$label")
  fi
}

echo "=== python tests ==="
for t in tests/test-*.py; do run_one "$(basename "$t")" python3 "$t"; done
echo "=== node tests ==="
for t in tests/test-*.mjs; do run_one "$(basename "$t")" node "$t"; done
echo "=== shell tests ==="
for t in tests/test-*.sh; do
  [ "$(basename "$t")" = "run-all.sh" ] && continue
  run_one "$(basename "$t")" bash "$t"
done

echo
echo "=== summary: $pass passed, $fail failed ==="
if [ "$fail" -ne 0 ]; then
  printf '  failed: %s\n' "${failed[@]}"
  exit 1
fi
exit 0
```
(Note: `tests/test-*.sh` glob — if there are non-test `.sh` files in tests/, the `run-all.sh` self-exclude handles itself; verify no other non-test .sh exists with `ls tests/*.sh`.)

- [ ] **Step 2: Make it executable + run it**

Run: `chmod +x tests/run-all.sh && bash tests/run-all.sh`
Expected: per-file `ok` lines + `=== summary: 48 passed, 0 failed ===`, exit 0. (Count = 43 py + 5 mjs + 0-or-more sh; confirm the actual total matches `ls tests/test-*.py tests/test-*.mjs tests/test-*.sh | grep -v run-all | wc -l`.)

- [ ] **Step 3: Verify it FAILS when a test fails** (the exit-code contract)

Run: temporarily break a trivial test and confirm non-zero exit:
```bash

git checkout tests/test-no-codex-leftovers.py
```
Expected: the summary shows `1 failed`, `exit=1`, and the test is named in the failed list. Then the `git checkout` restores it.

- [ ] **Step 4: Commit**

```bash

git commit -m "test(ci): add tests/run-all.sh entry-point (full offline suite, exits non-zero on failure)"
```

### Task 3: GitHub Actions workflow (B3)

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: `tests/run-all.sh` (Task 2).
- Produces: a workflow that runs the suite on PR + push to main, offline, no secrets.

- [ ] **Step 1: Write `.github/workflows/ci.yml`**

```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.x"
      - uses: actions/setup-node@v4
        with:
          node-version: lts/*
      - name: Run test suite (offline, mock mode)
        run: bash tests/run-all.sh
```
(No `env:` with secrets — the suite runs in mock mode with no DeepSeek/network. No rust/deno setup — the project is python+node only.)

- [ ] **Step 2: Validate the yaml**

Run: `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml')); print('yaml ok')"`
Expected: `yaml ok`. (If `actionlint` is installed, also run it; not required.)

- [ ] **Step 3: Confirm the workflow runs the suite locally** (the step's command works)

Run: `bash tests/run-all.sh >/dev/null 2>&1 && echo "suite green (CI would pass)" || echo "suite red"`
Expected: `suite green (CI would pass)`.
