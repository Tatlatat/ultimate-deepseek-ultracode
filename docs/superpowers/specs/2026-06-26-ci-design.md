# Sub-project B — CI — Design

**Date:** 2026-06-26
**Status:** Design (approved in brainstorming, pending spec review)
**Goal:** Add CI so every PR/push runs the test suite + guard automatically — Sub-project B of the public-OSS-ship effort (A refactor done; B CI; C docs+flags; D push).

## Measured facts (the basis)
- No CI exists (no `.github/workflows/`).
- Test suite: 43 python `test-*.py`, 5 node `test-*.mjs`, 4 shell `*.sh`.
- **42/43 python + 5/5 node + 4/4 shell pass OFFLINE** (mock mode, no DeepSeek/network). Only `test-gateway-nonstream-heartbeat.py` is flaky.
- Benches (realworld/cross-workflow/decompose-ab/lever-matrix/capability) need REAL DeepSeek → NOT in CI.
- Heartbeat flakiness root cause: `test_heartbeat_fires_before_slow_producer_returns` uses a real `time.sleep(2.5)` producer block with a 1s keepalive interval and asserts `>=1 heartbeat` — a wall-clock race that slips on slow/loaded CI runners (one run measured 21.5s; under load it can hang/fail).

## Components

### B1 — De-flake the heartbeat test (so it can go in CI)
**Where:** `tests/test-gateway-nonstream-heartbeat.py`, `test_heartbeat_fires_before_slow_producer_returns`.
**What:** widen the timing margin so a heartbeat is overwhelmingly likely even on a slow runner: set the keepalive interval SMALL (e.g. `CLAUDE_REASONIX_GATEWAY_STREAM_KEEPALIVE_SECONDS=0.2`) and the producer block LONGER (e.g. `block_secs=2.0`) — expected ~10 heartbeats, so even if CI drops most, `>=1` holds. Keep the assertion (`>=1 content_block_delta`) and the test's MEANING (a slow non-stream producer must emit heartbeats before it returns). Do NOT weaken the assert; widen the margin that feeds it. The other two heartbeat tests in the file (non-stream emits heartbeat; stream=true still works) are NOT timing-raced (no sleep) — leave them.
**Done when:** the test passes deterministically across repeated runs (run it 5x back-to-back, all pass), and a single run is faster (shorter block where possible without re-introducing the race).

### B2 — One test entry-point `tests/run-all.sh`
**Where:** new `tests/run-all.sh`.
**What:** a single script that runs the FULL CI-safe suite and exits non-zero on any failure:
- every `tests/test-*.py` via `python3`
- every `tests/test-*.mjs` via `node`
- every `tests/test-*.sh` via `bash` (excluding `run-all.sh` itself)
- prints a per-file PASS/FAIL line + a final summary (`N passed, M failed`).
- EXCLUDES the benches (they're in `runtime/`, not `tests/`, so already excluded) and any test that genuinely needs DeepSeek. After B1, the heartbeat test is included. If any test needs network and has no mock, it is named in an explicit SKIP list with a one-line reason (currently none — all 48 pass offline after B1).
- runnable identically locally and in CI: `bash tests/run-all.sh`.
**Done when:** `bash tests/run-all.sh` runs all 48 tests, prints the summary, exits 0 when green / 1 when any fail (verified by temporarily breaking one).

### B3 — GitHub Actions workflow `.github/workflows/ci.yml`
**Where:** new `.github/workflows/ci.yml`.
**What:** a minimal, dependency-light workflow:
- Triggers: `pull_request` + `push` to `main`.
- Runs on `ubuntu-latest`.
- Steps: checkout → setup-python (3.x) → setup-node (lts) → `bash tests/run-all.sh`.
- NO DeepSeek secret, NO network calls (CI runs in mock mode). NO rust/deno (the gateway is python+node only; the deno_lint/bun work was validation scratch, not the project).
- The guard `tests/test-no-codex-leftovers.py` is part of run-all, so it gates every PR.
**Done when:** the yaml is valid (`python3 -c "import yaml; yaml.safe_load(...)"` or actionlint if available), the steps reference only what the suite needs, and the workflow would run `tests/run-all.sh` on a PR.

## How we'll know it worked
1. The heartbeat test passes 5/5 repeated runs (de-flaked).
2. `bash tests/run-all.sh` runs all 48 tests, exits 0 green / 1 on any failure.
3. `.github/workflows/ci.yml` is valid and runs run-all.sh on PR + push to main, with no secrets/network.

## Out of scope
- No bench in CI (needs real DeepSeek).
- No docs/README/flag-hiding (Sub-project C) and no version/tag/push (Sub-project D).
- No new test coverage — CI runs the EXISTING suite (plus the de-flaked heartbeat test).
- No change to gateway/engine/levers behavior.
